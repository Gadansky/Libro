# Instrucciones del proyecto

Este repositorio contiene la documentacion de canon, estructura narrativa y backlog de una novela.

## Uso de Fiction Context

- Para preguntas narrativas sobre canon, continuidad, personajes, artefactos, lugares, arcos, pendientes, plantas/payoffs o "que tengo definido hasta ahora", usar primero el MCP `Fiction Context`.
- Herramientas de `Fiction Context`:
  - `search_bible`: busqueda general en canon, biblia, arcos, personajes, artefactos, lugares y notas.
  - `get_character`: informacion de personajes y estado por capitulo cuando aplique.
  - `get_foreshadowing`: plantas, payoffs, causalidad, pendientes narrativos y conexiones futuras.
  - `get_chapter_context`: contexto de capitulo cuando la pregunta sea sobre un capitulo especifico.
  - `check_continuity`: revision auxiliar de continuidad de pasajes; no tratarla como veredicto absoluto ni como reemplazo del criterio narrativo.
- Si `Fiction Context` no encuentra suficiente contexto, no cubre la pregunta, o hace falta texto exacto con lineas, usar los archivos locales del repositorio como respaldo.
- Si se responde principalmente desde archivos locales en vez de `Fiction Context`, indicarlo brevemente cuando sea relevante.
- No usar busqueda web para el canon del proyecto salvo que el usuario lo pida explicitamente.

## Jerarquia documental

- `Biblioteca/01_Biblia_de_mundo.md`: reglas de mundo y canon estable.
- `Biblioteca/03_Arcos_narrativos.md`: acontecimientos, estructura narrativa y revelaciones.
- `Biblioteca/04_Personajes.md`: fichas de personajes y entidades.
- `Biblioteca/05_Lugares_facciones_y_sociedades.md`: geografia, instituciones, lugares y sociedades.
- `Biblioteca/07_Artefactos.md`: catalogo operativo de artefactos.
- `Biblioteca/08_Backlog_de_ideas.md`: pendientes e ideas no cerradas.
- `Biblioteca/09_Diagnostico_y_plan_de_mejora.md`: diagnostico, lagunas y plan documental.
- `Biblioteca/99_Legacy.md`: documento historico; no usar como canon principal salvo que otro archivo lo haya canonizado.

## Estilo de respuestas

- Responder en espanol, salvo que el usuario pida otro idioma.
- En respuestas de canon, separar canon definido, inferencias razonables y pendientes marcados.
- Citar archivos y lineas cuando ayude a ubicar la fuente.
- No modificar documentos de canon sin una instruccion explicita del usuario.
